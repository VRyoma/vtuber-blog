# VTuber ネタ帳 — 下書きのブラッシュアップ

`draft: true` のまま寝ている記事や、雑に書いた本文を **公開品質** に仕上げるスキル。

## 起動条件

- 「〈slug〉を仕上げて」「この記事をブラッシュアップ」「公開品質に」
- `/vtuber-post-polish` が入力された

## 事前準備

1. `CLAUDE.md` を Read
2. 対象ファイル `content/ja/post/<slug>/index.md` を Read
3. ワードカウント、H2 数、具体例の有無を把握

## チェックリスト

順に適用する。各項目で修正が入ったらユーザに要点を報告する。

### A. フロントマター

- [ ] `title` が 32 字以内。長いなら 2 案提示
- [ ] `description` が 120 字以内で、**第一文で結論**を述べている
- [ ] `categories` は 1 つだけ
- [ ] `tags` が 2〜5 個。重複語彙の統一 (例: `初見` → `初見向け`)
- [ ] `date` が不正な値でないか
- [ ] `draft: true` であれば公開準備が終わってから `false` に

### B. 本文トーン

- [ ] **「〜が重要です」で締める段落** を検出し、直後に「具体的には〜」を追加
- [ ] 一方的な断定 (「絶対」「必ず」「すべて」「完全に」) を柔らかい表現に
- [ ] 主語の大きい表現 (「みんな配信者は」「○○勢は」) を主語を絞って書き直す
- [ ] 一文が 60 字を超える箇所を分割

### C. 具体性

- [ ] 各 H2 に **数字入りの具体例** が 1 つあるか
- [ ] 抽象名詞 (「工夫」「意識」「姿勢」) が続く段落に具体アクションを追加
- [ ] 読者が「今日試せる」アクションが末尾に最低 1 つ

### D. 他者配慮 (CLAUDE.md の方針遵守)

- [ ] 個別 VTuber や事務所への **評価・比較・批判** が紛れていないか (要 grep: 「〜より」「〜と違って」)
- [ ] 出所不明の業界情報 / 噂を削除
- [ ] 違法・規約違反を誘発する表現 (「無断で」「バレなきゃ」等) を削除

### E. 構造

- [ ] リードが 150〜300 字
- [ ] H2 が 3〜5 個 (極端に少ない/多い場合は再構成を提案)
- [ ] まとめに要点 3 つの箇条書き
- [ ] 関連記事リンクが貼れそうなら提案 (既存記事を Glob で確認)

### F. SEO 補強

- [ ] `title` に狙いキーワードが含まれる
- [ ] H2 見出しにもキーワードが自然な形で散らばる
- [ ] `description` にキーワードを 1 回入れる (多用は逆効果)

## 仕上げの手順

1. 上記チェックリストを一気通貫で適用
2. 修正後の記事全体を diff で示す (大きい変更は要点サマリでも可)
3. `hugo --gc --minify` を実行し warning/error 0 を確認
4. ユーザへの最終報告:
   - 変更点サマリ (3〜5 行)
   - 修正後のワードカウント
   - プレビュー URL
   - 公開前の人力確認ポイント (画像差し替え、誤字再読、固有名詞)

## 禁止

- ユーザの意図を大きく変える構成の差し替えを **相談なしに** やる
- 下書きに含まれる実在 VTuber 名を勝手に消す (意図的な言及かもしれない → 確認)
- `draft: true` → `false` をユーザ確認なしに行う
