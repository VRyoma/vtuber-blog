「配信中に何を話せばいいか分からなくなる」——そんな経験をしたことはありますか。アドリブで乗り切れる日もあれば、10 分沈黙してしまう日もある。台本を用意したくても、毎回一から書くのは時間がかかります。

AI に「配信の種類・今日の気分・視聴者へ聞きたいこと」を渡すだけで、10 分で使える進行メモが出来上がります。この記事ではそのプロンプトと実例を紹介します。

**この記事で分かること**
- 台本なしで起きがちな失敗と解決の考え方
- 雑談・ゲーム実況・歌枠それぞれの AI 台本プロンプト
- 出てきた台本を自分らしく仕上げる 3 つの編集ポイント

---

## 「アドリブ配信」が崩れやすい 3 つの場面

「自分はトーク力があるから台本は不要」と感じていても、特定の場面ではアドリブが難しくなります。崩れやすい 3 か所を知っておくと、台本が必要な部分だけ用意できます。

### 配信開始直後の 5 分が特に危険な理由

配信開始直後は、視聴者の人数が少なくコメントも少ない状態です。「え、えーと今日は〜」と始まると、その後の配信全体のテンションが上がりにくくなります。

「開始の一言・今日やること・視聴者への最初の質問」の 3 点をメモするだけで、スタートが安定します。

### ゲームの待機・ロード中・エンディングの空白

ゲーム実況では、ロード画面やゲームオーバー後の待ち時間が不規則に発生します。この 10〜30 秒の沈黙が続くと、視聴者が離脱しやすくなります。「ロード中に話す 3 つのネタ」を事前に用意しておくだけで、この空白を埋められます。

### 配信の締めで何も言えなかったパターン

エンドカードに切り替わる直前、「え、終わり？　ありがとうございました」と終わってしまうケースは少なくありません。締めは台本がいちばん効果を発揮する部分です。「次回告知・感謝・視聴者へのひと言」の 3 行を書いておくだけで、配信の印象が大きく変わります。

## AI 台本プロンプトの基本構造

台本を作るプロンプトには「配信タイプ・今日の目玉・視聴者への問い」の 3 点を入れるのが基本です。この 3 点を渡すだけで、AI は 1〜2 分で進行メモの原型を返してくれます。

### 基本テンプレート

次のプロンプトをそのままコピーして、`[ ]` の中を埋めるだけです。

```
あなたは VTuber 配信のアシスタントです。
以下の情報をもとに、今日の配信の進行メモを作ってください。

・配信タイプ: [雑談 / ゲーム実況 / 歌枠 / その他]
・今日の目玉: [例: 新しいゲームの初見プレイ]
・視聴者に聞きたいこと: [例: 最近ハマっているゲームは？]
・配信の長さ: [例: 1 時間]

出力形式:
1. 開始トーク（3〜4 行）
2. メインコンテンツの進行ポイント（箇条書き 5〜7 点）
3. つなぎセリフ候補（2〜3 個）
4. 締めトーク（3〜4 行）
```

### プロンプトの読み方と使いどころ

AI が返した出力は「そのまま読む台本」ではありません。「今日の配信で触れたいポイントの一覧表」として使います。出力を受け取ったら次の 2 点を確認しましょう。

- 自分が実際に言いそうな言葉になっているか
- 視聴者への質問が 1 か所以上入っているか

この 2 点を満たしていれば、あとは配信中にちらちら見ながら進めるだけです。

## 配信タイプ別プロンプト実例

3 つの配信タイプに合わせたプロンプトをそのまま使える形で紹介します。`[ ]` 部分を書き換えてください。

### 雑談配信用

```
あなたは VTuber 配信のアシスタントです。
今日は 1 時間の雑談配信をします。

テーマ: [例: 最近買ったもの自慢]
視聴者に聞きたいこと: [例: 最近の衝動買いは何ですか？]
話したい小ネタ: [例: コンビニで変なものを見つけた話]

以下を作成してください。
1. 開始の一言（フレンドリーに、1〜2 文）
2. 雑談の話題展開（3〜4 ステップ）
3. 視聴者コメントを拾うタイミングの目安
4. 締めの一言
```

### ゲーム実況用

```
あなたは VTuber 配信のアシスタントです。
今日は [ゲーム名] を実況します。

プレイ状況: [例: 初見 / 2 周目 / ボス戦のみ]
見せ場になりそうな場面: [例: ステージ 3 のボス]

以下を作成してください。
1. ゲーム紹介トーク（知らない人にも分かるように、2〜3 文）
2. プレイ中に話すポイント（5 点、各 1 行）
3. ロード・待機中に話せるネタ（3 個）
4. エンドトーク（次回予告含む）
```

### 歌枠用

```
あなたは VTuber 配信のアシスタントです。
今日は [〜分] の歌枠をします。

テーマ: [例: アニソン縛り / 懐かし J-POP]
セットリスト: [曲名を 3〜5 曲]

以下を作成してください。
1. 歌枠開始トーク（30 秒程度）
2. 各曲の前後に使える MC コメント（1 曲につき 1〜2 行）
3. リクエストを受け付けるタイミングの案内文
4. 締めのあいさつ
```

## AI 台本を「自分らしく」仕上げる 3 つの編集

AI が生成した台本をそのまま読むと棒読みになりやすいです。3 つのポイントで手直しすると自分の言葉に変わります。

### ① 語尾・口癖をひと通り置き換える

AI は丁寧な書き言葉で台本を作ります。自分が普段配信中に使う語尾（「〜だよね」「〜って感じ！」など）に置き換えましょう。3〜5 か所直すだけで格段に読みやすくなります。

### ② 視聴者への質問は 1 か所だけ残す

AI は複数の「視聴者に聞いてみましょう」を挿入しがちです。1 時間の配信で質問コーナーが 5 か所あると流れが途切れます。最も盛り上がりそうな 1 か所だけ残して、他は削除しましょう。

### ③ 台本は「見出しだけ使う」くらいの気持ちで

生成された台本の本文を一字一句読もうとすると、画面の外を見ながら配信することになります。実際には「開始・ゲーム紹介・ボス前・締め」という見出しだけ目に入れば十分です。

| 項目 | AI の出力（編集前） | 自分用に直した版 |
|---|---|---|
| 開始 | 「みなさん、こんにちは！今日もよろしくお願いします」 | 「おはよ〜！来てくれてありがとね」 |
| 締め | 「以上で本日の配信を終了します」 | 「今日も来てくれてありがとー、また来てね！」 |

---

## 台本を使い続けるための 2 分フィードバック

配信後に 2 分だけ、次のメモを AI に渡します。これを繰り返すと、5〜6 回で自分の配信スタイルに合ったテンプレートが育ちます。

```
前回の台本で「使えた」部分: [例: ゲーム紹介トークは自然に言えた]
「使えなかった」部分: [例: ロード中のネタが長すぎた]
次回に変えてほしい点: [例: ロード中ネタは 1 個に絞って短くする]
```

## まとめ

- **アドリブが崩れやすいのは「開始・待機・エンディング」の 3 場面**。この 3 か所だけ台本を用意すれば十分
- **プロンプトに「配信タイプ・目玉・視聴者への問い」を渡す**と、10 分以内に原型ができる
- **AI 台本は「見出しだけ使う」**くらいの気持ちで。自分の口癖に直してから配信に臨もう
