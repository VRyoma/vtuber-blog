配信中に「あ、BGM変えなきゃ」「シーン切り替え忘れた」と慌てた経験ありませんか。

OBS のシーンやソースが増えすぎると、目的のシーンを探すだけで数秒かかってしまい、配信のテンポが崩れます。コメントを読む余裕もなくなり、結果的に配信品質が下がってしまいます。

この記事では、シーンとソースを整理して、**ホットキー一発で3秒以内に切り替えられるようにする手順**を紹介します。

**この記事で分かること:**
- シーンとソースを減らして迷わなくする整理法
- ホットキーの割り当て方とおすすめの組み合わせ
- 配信中によくある切替ミスを防ぐ具体的な設定

---

## シーンが増えすぎる原因と「7シーン以内」に絞る理由

OBS を使い続けていると、企画ごとにシーンを作ったり、コラボ用に追加したりして、いつの間にか10個以上のシーンが並んでいませんか。

シーンが10個を超えると、**目的のシーンを探すのに2〜3秒**かかるようになります。配信中の2〜3秒は、コメントを見落としたり、沈黙が生まれたりするのに十分な時間です。

### 7シーン以内に収めるための見直し手順

まずは現状のシーン一覧を書き出してみてください。「いつも使うシーン」と「月に1回以下しか使わないシーン」に分けます。

**基本の7シーン構成例:**

| # | シーン名 | 用途 |
|---|---|---|
| 1 | 待機 | 配信開始前・終了後の「まもなく開始」画面 |
| 2 | ゲーム | ゲーム配信のメイン画面 |
| 3 | 雑談 | カメラメインの雑談画面 |
| 4 | 歌枠 | 音源+立ち絵の歌枠画面 |
| 5 | 休憩 | BRB（離席中）画面 |
| 6 | 終了 | 「ありがとうございました」画面 |
| 7 | 画面共有 | コラボ・Discord 画面共有用 |

月に1回以下のシーンは、**その都度作って配信後に削除する**運用がおすすめです。「コラボ相手別」のシーンも、画面共有シーンを1つ用意しておけば使い回せます。

### Before → After のイメージ

**Before (12シーン):**
コラボA、コラボB、ゲーム1、ゲーム2、雑談、歌枠、休憩、終了、待機、告知、テスト、画面共有

**After (7シーン):**
待機、ゲーム、雑談、歌枠、休憩、終了、画面共有

「告知」は雑談シーンのソースとして切り替える、「テスト」は本番プロファイルとは別に管理する、といった整理で7つに収まります。

## ソースの重複をなくす — 「共通ソース」の作り方

シーンを整理したら、次は**ソースの重複**をなくします。

同じカメラ枠やテロップを複数シーンにコピペしていると、修正のたびに全シーンを書き換える手間が発生します。BGMの音量を変えただけで5つのシーンを直す、という状態から抜け出しましょう。

### よくある重複パターン

- **カメラ枠** — ほぼすべてのシーンで同じ位置・サイズ
- **BGM音源** — 待機・休憩・終了で同じ曲を使い回し
- **テロップ（名前・SNSリンク）** — 画面端の固定表示

これらを「共通ソース」として1箇所で管理すると、**修正は1回で済みます**。

### グループ化で一元管理する手順

1. OBS の「ソース」欄で、複数シーンにまたがるソースを右クリック
2. 「コピー」→ 別シーンで「参照（リンク）として貼り付け」を選択
3. 元のソースを編集すれば、リンク先すべてに反映される

```
例: カメラ枠を「雑談」シーンで作成
  → 「ゲーム」「歌枠」には「参照として貼り付け」
  → カメラの位置を変えたいときは「雑談」シーンの1箇所を直すだけでOK
```

**注意点:** 「コピー」と「参照（リンク）」は違います。普通のコピーは独立したソースになるため、片方を直してももう片方には反映されません。必ず「参照として貼り付け」を使いましょう。

## ホットキーを割り当てて3秒で切り替える

シーンとソースが整理できたら、次は**キーボード一発で切り替える設定**です。マウスでシーン一覧から探してクリックするのをやめ、ホットキーに登録します。

### おすすめのホットキー割り当て表

| 操作 | おすすめキー | 理由 |
|---|---|---|
| シーン: 待機 | F1 | 左端で押しやすい |
| シーン: ゲーム | F2 | 頻繁に使うので近くに |
| シーン: 雑談 | F3 | 同上 |
| シーン: 歌枠 | F4 | 同上 |
| シーン: 休憩 | F5 | 休憩は日常的に使う |
| シーン: 終了 | F12 | 誤爆を防ぐため遠いキーに |
| マイクミュート | Ctrl+M | 片手で押せる |
| デスクトップ音ミュート | Ctrl+D | 同上 |

### ホットキーの設定手順

1. OBS メニューの「設定」→「ホットキー」を開く
2. 「シーンの切り替え」欄に、各シーン名が並んでいる
3. 各シーンの入力欄をクリックし、割り当てたいキーを押す
4. 「OK」で保存

**ポイント:** F9〜F11 などは他のソフトと衝突しやすいので、F1〜F5 + F12 の組み合わせが無難です。ゲーム中にファンクションキーが使えない場合は、`Ctrl+数字` や `Alt+数字` に代替しましょう。

### ストリームデッキがない場合の代替

Elgato Stream Deck のような物理ボタンがなくても、**スマホをリモコン代わりにする**方法があります。「OBS Remote」や「Streamlabs Remote」などの無料アプリを使えば、スマホ画面にシーン切替ボタンを配置できます。机の上にスマホを置いておけば、マウスを動かさずにシーンを切り替えられます。

## 配信中の切替ミスを防ぐ3つの工夫

設定が整っても、配線中にうっかりミスは起きます。よくあるミスを**事前に防ぐ設定**を3つ紹介します。

### 1. シーン名に番号プレフィックスを付ける

シーン一覧がアルファベット順で並ぶと、目的のシーンが見つかりにくいです。**先頭に番号を付ける**と、いつも同じ順序で並び、迷いません。

```
01_待機
02_ゲーム
03_雑談
04_歌枠
05_休憩
06_終了
07_画面共有
```

番号順に並ぶことで、ホットキーとシーンの対応も覚えやすくなります。

### 2. ミュート解除忘れを「ソースミュート」で防ぐ

シーンを切り替えたときに「マイクがミュートのまま話してしまった」というミスはよくあります。これを防ぐには、**OBS の「ソースのミュート」ではなく「オーディオミキサー」で管理**するのがコツです。

具体的には:
- マイクのミュートは**ホットキー (Ctrl+M)** で切り替える
- シーンごとにミュート状態を引き継ぐ設定にする (OBS 設定 → オーディオ → 高度な設定)
- 配信開始前にマイクが「ミュート解除」状態であることを確認する

### 3. 配信前の3項目チェックリスト

配信開始前に以下を確認するだけで、切替ミスの大部分を防げます。

| # | チェック項目 | 確認方法 |
|---|---|---|
| 1 | マイクがミュート解除されている | オーディオミキサーのバーが動いているか |
| 2 | 最初のシーンが「待機」になっている | シーン一覧の「01_待機」が選択状態か |
| 3 | ホットキーが効く | 実際に F3 → F2 の順に押して切り替わるか |

3つのチェックは30秒で終わります。配信開始のルーティンに組み込んでみてください。

---

## まとめ

- シーンは**7つ以内**に絞り、使わないものは削除する
- 共通ソースを「**参照として貼り付け**」で1箇所管理し、重複コピペをやめる
- **ホットキー**を割り当てて、キーボード一発で切り替えられるようにする

次の配信の前に、まずはシーン一覧を見直してみてください。「いつも使う7つ」を残して整理するだけで、配線中の操作に余裕が生まれます。
