コラボしたいVTuberがいても「どう声をかければいいか分からない」「企画の説明がうまくできない」で打診を先延ばしにしていることはないでしょうか。

相手に企画書を送れる状態にしておくと、声をかけるハードルが一気に下がります。AIに自分の配信スタイルと相手のチャンネル情報を渡すだけで、相手に見せられる企画書の骨子が10分でできます。

## コラボ企画書があると打診が成功しやすい理由

### 相手が「参加するかどうか」を判断できる

「一緒にコラボしませんか」だけでは、相手はどんな内容か分からず返答しにくい状態です。企画書があると「配信時間は1時間、内容はゲーム対戦3本」のように具体的なイメージが伝わり、相手が検討しやすくなります。

### 自分の意図が正確に伝わる

口頭や短いDMでは「何のためのコラボか」が伝わりにくいです。企画書に「お互いのチャンネルの視聴者に届けたい」「この配信シーズンの締めくくりにしたい」などの目的を書いておくと、相手が企画の意図を理解した上で返答できます。

### 断られても次に活かせる

断られた場合も「このタイミングは難しいですが企画内容は面白そうです」のような返答が来ることがあります。企画書があると「何がNGだったか」が分かりやすく、次回の改善につながります。

## AIで企画書を作るプロンプト

以下のプロンプトを使います。`[ ]` を自分の情報に書き換えます。

### ステップ1：企画の骨子を作る

```
VTuberのコラボ企画書を作りたいです。
以下の情報をもとに、相手に送れる企画書の骨子を作ってください。

【自分の情報】
チャンネル名：[自分のチャンネル名]
配信ジャンル：[ゲーム実況 / 歌枠 / 雑談 など]
チャンネル登録者数の目安：[1,000人 / 5,000人 など]
視聴者の特徴：[ホラーゲーム好きが多い / 深夜配信が中心 など]

【相手の情報】
チャンネル名：[相手のチャンネル名]
相手の配信ジャンル：[ゲーム実況 / 歌枠 など]
コラボしたい理由：[視聴者層が似ている / 同じゲームをやっている など]

【コラボの内容イメージ】
やりたいこと：[ゲーム対戦 / 歌のセッション / 雑談配信 など]
配信時間の目安：[1時間 / 2時間 など]
配信場所：[お互いのチャンネルで同時配信 / どちらか一方のチャンネルで など]

以下の項目を含む企画書を作ってください：
・企画タイトル
・企画の目的（1〜2文）
・企画の内容・流れ（箇条書き）
・お互いのメリット
・配信日程の候補（仮）
・備考
```

### ステップ2：打診文を作る

```
この企画書をもとに、相手のDMに送る打診文を作ってください。
・200〜300字程度
・押しつけがましくなく、相手が断りやすい余地を残した文章
・企画書の内容を簡潔に紹介し、詳細は添付の企画書で確認してもらう形
```

## 企画書の出力例

### タイトル例

AIが提案してくれる企画タイトルのイメージ：

```
「初見ホラーゲーム対決！ビビりランキング決定戦」
「二人でつなぐ歌リレー企画 〜好きな曲を交互に歌う30分〜」
「雑談コラボ：お互いの視聴者に「実はこんなことやってます」紹介回」
```

タイトル案が気に入らなければ「もっと〇〇な雰囲気にして」と追加指示できます。

### 「お互いのメリット」の書き方

この項目があると相手が「参加する理由」をイメージしやすくなります。

```
例：
・AさんのチャンネルがBさんの視聴者に知られるきっかけになる
・BさんのチャンネルがAさんの視聴者に届く
・コラボ配信はアーカイブとして残るため、後から検索で見つけてもらえる
```

## 企画書の送り方

### PDF または Google ドキュメントで共有する

AIが作った企画書のテキストをGoogle ドキュメントに貼り付けてリンクを共有するか、PDFに書き出して添付します。DMで長文を送るより、ドキュメントのリンク1行の方が相手に読んでもらいやすいです。

### DM→企画書の順番で送る

先に「一度コラボの話をお聞きいただけますか」とDMを送り、返答をもらってから企画書を送ります。いきなり企画書を送ると相手に「読まなければいけない」という負担をかけます。返事があってから企画書を見せることで、相手が主体的に関わる状態を作れます。

### 断られたときの返し方

「今回は難しいです」と返ってきた場合は「また機会があればぜひ」で終わらせます。断りを引き留めたり理由を聞いたりすることは相手に負担をかけます。企画書は次回の打診時に改訂版として使い回せます。

---

## まとめ

- **企画書があると相手が判断しやすく、打診の成功率が上がる**。口頭やDMだけでは伝わらない「目的・流れ・メリット」が一目で分かる形にできる
- **プロンプトは2ステップ**：企画の骨子を作るプロンプト→打診文を作るプロンプトの順に使う
- **DM→企画書の順番で送る**。最初から企画書を押しつけず、相手が主体的に読む状況を作ってから共有する
