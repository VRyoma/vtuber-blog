コラボ実況の当日に「音が聞こえない」「どっちがホストか分からない」「ゲームのルールの認識がズレていた」——こうしたトラブルのほとんどは、事前に 10 分話し合えば防げます。

コラボ相手との合わせを後回しにすると、当日の配信が「問題解決の時間」になってしまいます。7 項目のチェックリストで、当日は配信に集中できる状態を作ります。

**この記事で分かること**
- コラボ実況でよくある当日トラブルの原因
- 事前合わせで確認する 7 項目
- 確認のタイミングと進め方

---

## コラボ当日のトラブルはほぼ「確認漏れ」

### 技術系のトラブル

マイクの音が相手に聞こえない、ゲーム音が大きすぎて声が埋もれる、Discord の通話品質が悪い——これらは「配信開始前にテストすれば分かった」問題です。配信開始後に気づくと、視聴者の前で無言のまま設定変更をすることになります。

### 進行系のトラブル

「どちらが司会をやるか」「ゲームオーバー時にどうするか」「何時間やるか」が決まっていないと、お互いが様子を見て沈黙が続きます。コラボ配信は 1 人配信より気を遣う場面が多いため、決めていない空白が増えやすいです。

---

## 事前合わせ 7 項目チェックリスト

### ① 配信の形式と主催

```
□ どちらのチャンネルで配信するか（または両方同時配信か）
□ アーカイブの扱い（両者のチャンネルに残すか）
□ 切り抜き・クリップの許可範囲
```

「両方のチャンネルで同時配信」の場合、URLが 2 本になります。視聴者が分散するため、告知でどちらを案内するか事前に決めます。

### ② 音声・技術環境の確認

```
□ 通話ツール（Discord / Zoom など）と部屋名
□ 互いの音声が聞こえるかテストする（配信 30 分前推奨）
□ ゲーム音・マイク音のバランス確認
```

技術テストは配信の「30 分前」が理想です。問題が出ても修正できる時間があります。

### ③ ゲームのルールと進め方

```
□ 使用するタイトル・モード・バージョン
□ ゲームオーバー時・詰まったときの対応（リセットする？諦める？）
□ 視聴者参加の有無（コメントをどう拾うか）
```

「私はクリア優先、相手は雑談優先」というズレが出ると、配信中にテンポが合わなくなります。

### ④ 配信の時間設計

```
□ 開始時刻・終了目安
□ 延長する場合の判断基準（お互いの体力・スケジュール）
□ 締め方の役割分担（どちらが締めの挨拶をするか）
```

終わり時間を決めておかないと、どちらも「相手が締めを切り出すのを待つ」状態になりがちです。

### ⑤ トーク・進行の役割分担

```
□ 「どちらがリード役か」の大まかな合意
□ 沈黙が続いた場合の対応（どちらが話題を振るか）
□ 禁止ネタ・触れてほしくないことがあれば共有
```

「禁止ネタ」の共有は特に重要です。配信中に意図せず相手の嫌なことに触れると、その場の空気が変わります。

### ⑥ 告知の形式

```
□ 告知用のポストは両者が出すか・どちらかが出すか
□ 告知内容（タイトル・出演者の表記）のすり合わせ
□ X や Discord でのリポスト・拡散の依頼有無
```

告知の文章が互いにズレていると、視聴者が「どの情報が正しいか」迷います。

### ⑦ トラブル時の連絡手段

```
□ 配信中に問題が起きたとき、すぐ連絡できる手段（DM・通話など）
□ 相手が突然落ちた場合の対応（何分待つか・どう視聴者に伝えるか）
```

配信中に相手の接続が切れても、連絡手段があれば状況を共有できます。「5 分待って連絡なければソロで続ける」など、具体的な基準を決めておくと動けます。

---

## 確認のタイミングと進め方

| タイミング | 確認内容 |
|---|---|
| 配信 2〜3 日前 | ①〜④⑤⑥ のすり合わせ（15 分の通話 or メッセージ） |
| 配信 30 分前 | ② の技術テスト + ⑦ の連絡手段確認 |
| 配信 5 分前 | 開始 OK の確認・挨拶 |

2〜3 日前に一度話しておくことで、当日は技術テストと最終確認だけで済みます。

---

## まとめ

- **コラボのトラブルのほとんどは事前確認の漏れ**。配信形式・技術環境・ルール・時間・役割・告知・緊急連絡の 7 項目を配信前に確認する
- **技術テストは 30 分前に行う**。音声・通話・ゲーム音のバランスを確認する時間を確保するだけで、当日の無言の設定変更がなくなる
- **「禁止ネタ」と「締め方の担当」を決めておく**。進行の空白と気まずい空気の 2 つを防ぐ最小限の合わせがこの 2 点
